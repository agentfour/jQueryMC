jQuery Model Controller Framework

jQuery Model Controller Framework

A simple framework that allows flexible yet structured JavaScript across a Multi-Page Application (MPA)

 

What is jQueryMC?

jQueryMC is a lightweight framework that is used for organising jQuery code into Models and Controllers of a web application's Front-End.

Specifically, Model code is most commonly AJAX Queries connecting to an API and is separated as such. Controllers is typically code that handles the operations for a given page, that might leverage Models and interact with the View. In this case, the View is the static output of an HTML page (even if it comes from a Dynamic Backend Source)

Why jQueryMC?

jQuery is a fantastic tool for JavaScript developers, however there is no common or standardised structure in the "Way to do things" with it.

There are other frameworks out there, specifically designed around the MVC (Model View Controller) design pattern. However, what if you don't want JavaScript to manage the view component of a site? There are a number of reasons you don't want JavaScript to wholly manage the View component of a site, such as:

  • Search Engine Optimisation

There are other benefits to using this framework too:

  • Lightweight (<3kb minified)
  • Flexible: You can use any existing library or plugin
  • Structured: Laid out in a typical MVC pattern, without the "V"

How to use?

First you need to set up the application:

  1. Download the source: git clone git@github.com:agentfour/jQueryMC.git
  2. Run NPM Install: npm install

A controller typically works with one or more views. So to tell the framework that you want a Controller to run within a specific view, on the HTML of the view, enter in the following code to trigger a controller to run:
<meta id="jmc-ctrl" jmc-ctrl="index" />

Breaking down this code, we are creating a naturally invisible element on the page (the META tag), and giving it an ID of jmc-ctrl. This tells the framework to look here for the name of the controller to run. Then to specify which controller to run, you have the code jmc-ctrl="index" which tells the framework to execute the index controller.
